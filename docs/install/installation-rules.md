# 安装规则说明

本文档包含 VibeSkills 安装过程中必须遵守的核心规则。这些规则适用于所有安装提示词。

---

## 📋 核心规则列表

### 规则 1-2：确认宿主和版本

**规则 1**: 在明确回答目标宿主之前，不要开始安装。

**规则 2**: 在明确回答公开版本之前，不要开始安装。

**说明**: 安装助手必须先询问用户要安装到哪个宿主（codex 或 claude-code），以及要安装哪个版本（全量版本或框架版本），然后才能开始执行安装命令。

---

### 规则 3-4：验证宿主和版本

**规则 3**: 如果回答的宿主不是 `codex` 或 `claude-code`，请直接告诉用户：当前版本暂不支持该宿主安装，并停止继续伪装安装。

**规则 4**: 如果回答的公开版本不是"全量版本+可自定义添加治理"或"仅核心框架+可自定义添加治理"，请直接告诉用户：当前公开安装说明暂不支持该版本名，并停止继续伪装安装。

**说明**: 只支持明确的宿主和版本选项，不支持的选项应该明确拒绝。

---

### 规则 5：系统判断

**规则 5**: 先判断当前系统是 Windows 还是 Linux / macOS，并使用对应命令格式。

**说明**:
- Linux / macOS 使用 `bash` 命令
- Windows 使用 `pwsh` 命令

---

### 规则 6：版本映射

**规则 6**: 将公开版本名映射到实际的 profile：
- "全量版本+可自定义添加治理" → `full`
- "仅核心框架+可自定义添加治理" → `framework-only`

**说明**: 用户看到的是友好的版本名，但实际安装时需要映射到技术 profile 名称。

---

### 规则 7-8：宿主特定配置

**规则 7**: 如果选择 `codex`：
- Linux / macOS 使用 `bash ./scripts/bootstrap/one-shot-setup.sh --host codex --profile [PROFILE]`
- 然后执行 `bash ./check.sh --host codex --profile [PROFILE] --deep`
- Windows 使用对应的 `pwsh` 命令
- 明确告诉用户：由于兼容性问题，当前版本暂不为 Codex 安装任何 hook
- 只围绕 Codex 当前可公开证明的本地 settings、MCP 和 CLI 依赖给建议

**规则 8**: 如果选择 `claude-code`：
- Linux / macOS 使用 `bash ./scripts/bootstrap/one-shot-setup.sh --host claude-code --profile [PROFILE]`
- 然后执行 `bash ./check.sh --host claude-code --profile [PROFILE] --deep`
- Windows 使用对应的 `pwsh` 命令
- 明确告诉用户：这只是 preview guidance，不是 full closure
- 明确告诉用户：由于兼容性问题，当前版本暂不为 Claude Code 安装 hook，也不再写 `settings.vibe.preview.json`

---

### 规则 9：密钥安全

**规则 9**: 对 `codex` 和 `claude-code`，都不要要求用户把密钥、URL 或 model 直接粘贴到聊天里；只告诉用户去本地 settings 或本地环境变量里配置。

**说明**: 保护用户的 API 密钥安全，不要让敏感信息出现在聊天记录中。

---

### 规则 10：Online Readiness

**规则 10**: 如果本地 provider 字段没有配置好，不能把环境描述成"已完成 online readiness"。

**说明**: 区分"本地安装完成"和"在线能力就绪"，不要误导用户。

---

### 规则 11：安装结果报告

**规则 11**: 安装完成后，请用简洁中文告诉用户：
- 目标宿主
- 公开版本
- 实际映射的 profile
- 实际执行的命令
- 已完成的部分
- 仍需手动处理的部分

**说明**: 提供清晰的安装结果摘要，让用户了解当前状态。

---

### 规则 12：真实性原则

**规则 12**: 不要把宿主插件、MCP 注册、provider 凭据伪装成已经自动完成；未补的项目优先表述成可选增强项或推荐下一步。

**说明**: 遵循 truth-first 原则，不要夸大安装的完成度。

---

### 规则 13：框架版本特别说明

**规则 13**: 对于"仅核心框架+可自定义添加治理"版本，还要额外明确告诉用户：当前拿到的是治理框架底座，不等于默认 workflow core 已经齐备；如果后续要接入自己的 workflow，请引导用户继续走 custom workflow governed onboarding。

**说明**: 框架版本和全量版本的交付物不同，需要明确说明。

---

## 📖 使用方法

在安装提示词中，可以这样引用这些规则：

```text
## 安装规则
详细规则请参考：[安装规则文档](../installation-rules.md)

核心规则摘要：
1. 必须先确认宿主（codex 或 claude-code）
2. 必须先确认版本（全量或框架）
3. 不支持的宿主/版本应明确拒绝
4. 判断系统类型，使用对应命令
5. 映射版本名到 profile
6-8. 执行宿主特定的安装命令
9. 不要让用户在聊天中粘贴密钥
10. 区分"安装完成"和"在线就绪"
11. 提供清晰的安装结果报告
12. 遵循 truth-first 原则
13. 框架版本需要额外说明
```
